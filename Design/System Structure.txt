# Technologies

### 1. 📱 Frontend (Desktop & Mobile)
*   **Dart**: Used with the **Flutter** framework. It is the primary language for building the user interface, handling the 60fps charting (via Cristalyse), and managing application state through the BLoC pattern.

### 2. 🧠 Backend & Trading Logic
*   **Python**: Recommended for **Strategy Development** and used by the **Hummingbot** framework. It is the industry standard for quantitative research and alpha generation due to libraries like Pandas and NumPy.
*   **C# (.NET)**: The native language of the **LEAN** engine (QuantConnect). It handles the high-performance core execution, accounting, and backtesting kernel.
*   **TypeScript**: Used specifically for the **Gateway DEX Middleware** (in the Hummingbot ecosystem) to standardize interactions across disparate blockchain networks.
*   **F#**: Mentioned as a supported functional language for authoring strategies within the LEAN framework.

### 3. 💾 Data & Infrastructure
*   **SQL**: Used for querying high-frequency tick data within **QuestDB**. The blueprint specifically highlights QuestDB's specialized SQL syntax (like `LATEST ON`) for low-latency lookups.
*   **q**: Mentioned as the highly specialized language for **kdb+**, though the blueprint favors QuestDB for modern, open-source deployments.
*   **Protobuf (Protocol Buffers)**: While not a programming language per se, it is the IDL (Interface Description Language) used for **gRPC** communication between the microservices.
*   **YAML / Shell**: For Docker configurations, Kubernetes manifests, and automation scripts.
